# RV Towing Calculator - 移动端优化报告

## 优化日期
2026年6月21日

## 优化目标
提升网站在移动设备上的用户体验，包括字体、显示、触摸、输入等各方面的便捷性和美观度。

## 优化内容

### 1. 全局 CSS 优化 (`src/app/globals.css`)

#### 添加的样式：
- **移动端输入框优化**：添加 `@media (max-width: 768px)` 媒体查询，强制设置输入框字体大小为 16px，防止 iOS 设备自动缩放
- **触摸目标优化**：为所有交互元素（button、a、input）设置最小高度和宽度（44px），符合移动端触摸标准
- **移除默认高亮**：添加 `-webkit-tap-highlight-color: transparent`，提升触摸体验
- **焦点优化**：为移动设备（无 hover 支持）优化焦点样式，使用 2px 轮廓

### 2. Header 组件优化 (`src/components/layout/Header.tsx`)

#### 优化内容：
- **汉堡菜单图标**：将文字按钮（"Menu"/"Close"）改为汉堡菜单图标（SVG），更直观且节省空间
- **移动端导航优化**：
  - 增大触摸目标：从 `py-3` 改为 `py-3`，并添加 `rounded-lg` 和 hover/active 状态
  - 改进布局：从 `flex flex-wrap gap-3` 改为 `grid grid-cols-1 gap-1`，使菜单项垂直排列，更易触摸
  - 增大字体：从 `text-sm` 改为 `text-base`
  - 添加分隔：在 Calculators 和 Guides 之间添加 `mt-6`，增加视觉分隔

### 3. 计算器组件优化

#### 优化的组件：
- `PayloadCalculator.tsx`
- `TowingCalculator.tsx`（包括 `NumberInput` 组件）
- `GvwrCalculator.tsx`
- `TongueWeightCalculator.tsx`
- `GcwrCalculator.tsx`

#### 优化内容：

##### a. 输入框优化：
- **增大 padding**：从 `px-3 py-2` 改为 `px-4 py-3`，增大触摸区域
- **增大字体**：在移动端使用 `text-base`（16px），在桌面端使用 `md:text-sm`
- **添加 inputMode**：为所有 number 输入框添加 `inputMode="numeric"`，在移动设备上显示数字键盘
- **优化焦点状态**：从 `focus:ring-1` 改为 `focus:ring-2`，使焦点更明显
- **增大标签间距**：从 `mb-1` 改为 `mb-2`，增加标签和输入框之间的间距
- **增大提示文字间距**：从 `mt-1` 改为 `mt-2`，增加提示文字和输入框之间的间距

##### b. 按钮优化：
- **增大 padding**：在移动端使用 `py-4`，在桌面端使用 `md:py-3`
- **增大字体**：在移动端使用 `text-lg`，在桌面端使用 `md:text-base`
- **优化 active 状态**：从 `active:scale-[0.99]` 改为 `active:scale-[0.98]`，提供更好的触摸反馈

##### c. Select 元素优化（仅 TowingCalculator）：
- **增大 padding**：从 `px-3 py-2` 改为 `px-4 py-3`
- **增大字体**：在移动端使用 `text-base`，在桌面端使用 `md:text-sm`
- **优化焦点状态**：从 `focus:ring-1` 改为 `focus:ring-2`

##### d. NumberInput 组件优化（TowingCalculator）：
- 应用与单个输入框相同的优化
- 统一输入框样式，确保所有计算器的一致性

### 4. 首页优化 (`src/app/page.tsx`)

#### 优化内容：

##### a. Hero 区域优化：
- **减小移动端 padding**：从 `py-20` 改为 `py-16`，在移动端减少留白
- **优化字体大小**：
  - 标题：在移动端使用 `text-3xl`，在桌面端使用 `md:text-5xl lg:text-6xl`
  - 描述：在移动端使用 `text-base`，在桌面端使用 `md:text-lg lg:text-xl`
- **增大按钮**：
  - 在移动端使用 `py-4`，在桌面端使用 `md:py-5`
  - 在移动端使用 `text-base`，在桌面端使用 `md:text-lg`
- **优化按钮间距**：从 `gap-3` 改为 `gap-4`，在移动端增加按钮间距

##### b. Calculators Grid 优化：
- **减小移动端 padding**：从 `py-16` 改为 `py-12`
- **优化标题字体**：在移动端使用 `text-2xl`，在桌面端使用 `md:text-3xl`
- **优化描述字体**：在移动端使用 `text-base`，在桌面端使用 `md:text-lg`
- **增大卡片 padding**：在移动端使用 `p-5`，在桌面端使用 `md:p-6`
- **优化卡片间距**：从 `gap-6` 改为 `md:gap-6`，在移动端使用默认间距
- **优化卡片标题**：在移动端使用 `text-base`，在桌面端使用 `md:text-lg`

## 优化效果

### 1. 触摸体验提升
- 所有交互元素的最小触摸目标达到 44x44px（Apple 推荐标准）
- 增大输入框和按钮的 padding，使触摸更准确
- 添加 active 状态反馈，提升触摸体验

### 2. 输入体验提升
- 输入框在 iOS 设备上不会自动缩放（字体大小 ≥ 16px）
- 数字输入框在移动端显示数字键盘（`inputMode="numeric"`）
- 增大输入框字体，提升可读性

### 3. 视觉体验提升
- 优化移动端字体大小，提升可读性
- 优化移动端间距和留白，提升视觉舒适度
- 统一所有计算器的样式，提升一致性

### 4. 导航体验提升
- 汉堡菜单图标更直观
- 移动端导航菜单的触摸目标更大，更易操作
- 改进移动端导航菜单的布局，减少误操作
